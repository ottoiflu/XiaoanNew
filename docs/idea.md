# 共享单车停放合规性判定标准

## 项目背景

为了确保共享车辆在还车过程中的正确摆放及满足合规性管理要求，本项目研发基于大模型和机器视觉的智能还车技术。系统将通过拍摄图片判断车辆位置、角度以及与周围环境的关系，确保车辆符合还车要求。

## 停车规范性判断维度

### 1. 图像构图合规性

**评测目的**：确保算法能获取有效信息。

**判断准则**：
- 完整性：车辆的关键部位（把手、坐垫、前后轮）必须在取景框内，无遮挡。
- 车牌可见性：车牌号码需清晰、完整，VLM 模型需能成功提取。
- 参考物包含：背景必须包含路面信息和停车基准线（如白线、马路牙子等）。确实没有明显参照则参考旁边车辆的停放朝向与趋势。

**打标选项**：
- `[合规]` 画面清晰，车辆大部分在框内且背景有能分辨的停车线或马路牙子。
- `[基本合规]` 车辆被截断大部分不在画面里但是有一类与二类参考线或周围其他车辆的停车趋势。
- `[不合规-构图]` 画面过暗/过曝、拍摄距离过远、车辆关键部位被遮挡。
- `[不合规-无参照]` 画面中未捕捉到第一、二类参考线标识、且也无相邻车辆方向参考。

### 2. 摆放角度合规性

**评测目的**：确保车辆与路面边缘垂直、与停车线垂直/平行，不占用多余空间。

**判断准则**：
- **基准线选择优先级**：
  1. 第一优先（标线参照）：若画面中有明确的地面停车标线（白线/黄线），则以标线为基准。
  2. 第二优先（几何参照）：若无标线，以马路牙子（路缘石）、墙边与地面接缝或花坛边缘、台阶边缘等明显的几何直线形状为基准。
  3. 第三优先（趋势参照）：若均无上述物体，参考地砖缝隙走向或周围车辆停放的统一趋势。
- **车辆向量定义**：以车辆中心轴线（后轮中点至把手中心）为向量。
- **合规要求**：
  - 第一类参考线：平行或垂直均可，偏差不超过 ±30°。
  - 第二类参考线：只判断是否垂直，偏差不超过 ±30°。

**打标选项**：
- `[合规]` 角度偏离在 30° 以内。
- `[不合规-角度]` 车辆严重斜停（偏离 ±30° 以上）。

### 3. 停放距离合规性

**评测目的**：确保车辆在指定的停放带内。

**判断准则**：
- 前轮边距：前轮触地边缘至停车边线的垂直距离 Df。
- 后轮边距：后轮触地边缘至停车边线的垂直距离 Dr。
- 合规要求：所有关键点距离基准线误差不得超过 5cm（基于深度估算）。

**打标选项**：
- `[完全合规]` 车辆整体处于停车线内且未压线。
- `[基本合规-压线]` 车辆大部分位于停车线内，出线部分小于一个轮胎。
- `[不合规-超界]` 车辆大部分位于停车线外，或离线距离远超一个车身的宽度。

### 4. 路面环境合规性

**评测目的**：通过语义识别判断路面性质。

**判断准则**：
- 停放介质：必须在硬化路面、透水砖面或指定停车格位。
- 禁止介质：严禁停在绿化带、车行道、消防通道、盲道。

**打标选项**：
- `[合规]` 停在指定路面。
- `[不合规-环境]` 停在非停车区域（如草坪、盲道）。

## 最终判定逻辑

四个维度中任一项出现"不合规"，最终判定为**不合规**。仅当所有维度均为合规或基本合规时，最终判定为**合规**。
